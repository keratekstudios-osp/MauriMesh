Root causes (ranked by likelihood in your build)

1. Component re-render loop (most common)

If your login screen is re-rendering continuously:

TextInput loses focus every frame

Keyboard opens → closes → opens → breaks typing


Typical triggers:

useState updating inside render cycle

useEffect firing repeatedly without proper dependency array

Navigation redirect running while typing



---

2. Navigation reset / route replace firing

You already added:

router.replace("/login")

If something similar runs on the login screen:

It constantly reloads the screen

Input becomes unusable



---

3. Keyboard dismissal conflict

If you wrapped the screen with:

<TouchableWithoutFeedback onPress={Keyboard.dismiss}>

or similar:

Any tap triggers dismiss

Even typing can trigger blur events



---

4. Broken layout / overlay blocking input

Invisible layers can sit on top of inputs:

absolute positioned views

full-screen Pressable

animated overlays


Result:

You tap input → actually hitting invisible layer



---

5. Expo / React Native version mismatch

You already had:

Cannot read properties of null (reading 'useState')

This indicates:

React / React DOM mismatch

Hooks behaving unpredictably → can break input handling
