# MauriMesh Replit Agent Final Rules

Understand first.
Protect the foundation.
Verify before change.
Use logic before action.
Push to perfect.
Respect original engineering.
Improve without destruction.
Speak facts.
Build for excellence.

The Replit Agent must always generate everything needed to complete an integration:

1. Structure
2. Architecture
3. Data contracts
4. Component wiring
5. UI wiring
6. State wiring
7. Error handling
8. Telemetry
9. Tests
10. Acceptance checks
11. Recovery path
12. Documentation

Every UI must have its intention defined and wired.

Every UI must look premium, expensive, calm, clean, and enterprise-grade.

MauriMesh must respect Māori protocol principles:

- Whakapapa: every packet, node, route, and decision must know its lineage.
- Manaakitanga: protect users and devices.
- Kaitiakitanga: protect the network from waste, spam, flooding, and harm.
- Rangatiratanga: preserve user control and local sovereignty.
- Kotahitanga: strengthen coordination between people and devices.

MauriMesh must include a self-healing software engineer layer.

When a fault is detected, the system must:

1. Identify the fault.
2. Protect working services.
3. Attempt safe recovery.
4. Log the decision.
5. Refuse unsafe automatic repairs.
6. Never fake success.
7. Report truth through telemetry.

A feature is only complete when:

- code exists
- wiring exists
- UI/state exists if user-facing
- telemetry exists
- error handling exists
- tests exist
- acceptance checks pass
- real-device verification is defined
